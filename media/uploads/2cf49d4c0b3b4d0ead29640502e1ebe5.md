# Shell 运维实战练习题（简化生产版）

## 1. 日志 5xx 告警统计
- 场景：文件 `access.log`（Nginx 样式）
- 要求：统计最近 10 分钟内 `5xx` 数量，并输出前三个最常见请求路径
- 目标输出示例：

```text
5xx_total=23
/top3_paths
/api/order 9
/api/pay 7
/favicon.ico 3
```

## 2. 服务健康检查脚本
- 任务：写 `health_check.sh`，检查 `nginx`、`docker`、`ssh` 三个服务是否在运行
- 要求：
  - 任一失败返回码非 0
  - 输出带时间戳
  - 结果同时写终端和 `/tmp/health_check.log`

## 3. 磁盘阈值巡检
- 任务：检查 `/`、`/var`、`/home` 分区
- 要求：
  - 使用率 > 80% 输出 `WARN`
  - 使用率 > 90% 输出 `CRIT`
  - 最后按严重级别汇总计数

## 4. 批量日志清理（安全版）
- 场景：目录 `/var/log/myapp` 下删除 7 天前 `.log` 文件
- 要求：
  - 先 dry-run，再真实删除
  - dry-run 只打印将删除的文件数与总大小
  - 真实执行需 `--force` 参数才允许

## 5. Nginx 配置发布脚本（简化 CICD）
- 任务：输入新配置路径
- 执行流程：备份当前配置 -> `nginx -t` 语法检查 -> reload
- 要求：任一步失败自动回滚，并输出每一步状态

## 6. Docker 容器巡检报表
- 任务：列出所有容器的以下字段：
  - 名称
  - 镜像
  - 状态
  - 重启次数
  - 端口映射
- 要求：
  - 输出为对齐表格
  - 单独列出 `Exited` 容器

## 7. API 可用性探测（带重试）
- 任务：探测 `https://example.com/healthz`
- 要求：
  - 最多重试 3 次，间隔 2 秒
  - 记录 HTTP code 和耗时
  - 连续失败则 `exit 2`

## 8. 简化版日志切割器
- 任务：对 `app.log` 做切割
- 规则：超过 50MB 就 rotate
- 要求：
  - 生成 `app.log.YYYYmmddHHMMSS`
  - 只保留最近 5 份，其余删除

---

你可以按 `1 -> 8` 顺序做；每做完一题，附上脚本和测试输出，再进行代码评审。
